# Публикация k2think-api-builder

## Подготовка к публикации

### 1. Проверка качества кода
```bash
# Запуск тестов
npm test

# Проверка линтинга
npm run lint

# Генерация документации
npm run docs
```

### 2. Сборка пакета
```bash
# Проверка файлов в пакете
npm pack --dry-run

# Создание пакета для тестирования
npm pack
```

### 3. Тестирование пакета
```bash
# Создайте тестовую директорию
mkdir test-package
cd test-package

# Установите локальный пакет
npm install ../k2think-api-builder-1.0.0.tgz

# Протестируйте импорт
node -e "const { CustomAPIBuilder } = require('k2think-api-builder'); console.log('✅ Пакет работает!');"

# Вернитесь назад
cd ..
rm -rf test-package
```

## Публикация в npm

### 1. Вход в npm
```bash
npm login
```

### 2. Проверка имени пакета
```bash
npm view k2think-api-builder  # Проверить, занято ли имя
```

### 3. Публикация
```bash
# Публикация последней версии
npm publish

# Или публикация beta версии
npm publish --tag beta
```

### 4. Проверка публикации
```bash
npm view k2think-api-builder
npm info k2think-api-builder
```

### 2. Обновление документации
- Обновите GitHub Pages если используется
- Проверьте, что документация доступна онлайн

### 3. Продвижение
- Опубликуйте анонс в социальных сетях
- Добавьте в списки npm пакетов
- Расскажите в сообществе

## Версионирование

Используйте Semantic Versioning:
- **Major (1.0.0)** - Обратные несовместимые изменения
- **Minor (1.1.0)** - Новые функции, обратно совместимые
- **Patch (1.1.1)** - Исправления ошибок, обратно совместимые

```bash
# Обновление версии
npm version patch  # 1.0.1
npm version minor  # 1.1.0
npm version major  # 2.0.0

# Публикация новой версии
npm publish
```

## Локальное тестирование

Для тестирования без публикации в npm:

```bash
# Создайте локальный npm registry
npx verdaccio

# Опубликуйте в локальный registry
npm publish --registry http://localhost:4873

# Установите из локального registry
npm install k2think-api-builder --registry http://localhost:4873
```
